# 単語登録機能の拡張計画書 (日英両表記対応)

本ドキュメントは、「単語を日本語表記と英語表記の両方登録できるようにしたい」という要望に基づき、システム改修を行うための計画書です。

## 1. 目的

専門用語を含む単語学習において、英語表記と日本語表記を明確に区別して管理・学習できるようにする。
入力時の混乱を防ぐため、UI上で明確なフィールドを用意する。

## 2. 変更内容

### 2-1. データモデル (Entity) の変更

`Word` エンティティにおいて、汎用的な `term` (用語) / `definition` (定義) というフィールド構成から、より具体的な `english` (英語) / `japanese` (日本語) という構成へ変更します。

- **変更前**:
  - `term`: String (単語そのもの)
  - `definition`: String (意味・定義)
- **変更後**:
  - `english`: String (英単語・英語表記)
  - `japanese`: String (日本語訳・日本語表記)
  - ※ `exampleSentence` (例文) はそのまま維持

### 2-2. データベーススキーマの更新

`spring.jpa.hibernate.ddl-auto=update` 設定により、Entityの変更に伴ってテーブル定義が自動更新されることを想定します。
既存データがある場合、カラム名の変更によりデータ移行が必要になる可能性がありますが、開発フェーズであるため、今回は新規カラム追加・旧カラム削除（または無視）の方針で進めます。

### 2-3. フロントエンド (Thymeleaf Template) の修正

- **登録/編集フォーム (`word_form.html`)**:
  - 入力欄を「英語 (English)」と「日本語 (Japanese)」に明確に分離。
  - どちらに何を入力すべきか迷わないラベル設定を行う。
  - プレースホルダーを活用し、入力例（例: "Vocabulary", "語彙"）を表示する。

- **一覧画面 (`word_list.html`)**:
  - テーブルのカラムを「英語」「日本語」に変更し、両方を並列で表示する。

### 2-4. バックエンド (Controller/Service) の修正

ControllerおよびService層におけるデータの受け渡し処理を、新しいフィールド名に対応させる。

## 3. 実装手順

1.  **Entityの修正**: `Word.java` のフィールド名を変更。
2.  **Repositoryの確認**: フィールド名変更に伴う修正（`test`などで使用していれば）。今回は標準的なCRUDのみのため影響は軽微と予想。
3.  **Templateの修正**: `word_form.html`, `word_list.html` のフィールドバインディングを修正。
4.  **動作確認**: アプリケーションを起動し、日英それぞれの登録・表示が正しく行えるか確認。

## 4. 検証項目

- [ ] 新規登録画面で「英語」「日本語」の入力欄が明確に分かれていること
- [ ] 英語、日本語の両方を入力して登録できること
- [ ] 一覧画面に英語と日本語が正しく表示されること
- [ ] 編集画面で既存の英語・日本語が正しくフォームにセットされ、更新できること
