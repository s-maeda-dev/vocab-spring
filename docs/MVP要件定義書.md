# 2. MVP要件定義書 (Minimum Viable Product)

## 2.1 MVPの目的

- ユーザー認証基盤の確立。
- ユーザー独自の単語データの保持。
- クイズ結果に基づいたGemini APIによるフィードバックの有効性を検証する。

## 2.2 実装範囲 (In Scope)

### Phase1. プロジェクトの初期化

### Phase2. 認証・ユーザー管理

- **基本ログイン機能**: ユーザー名とパスワードによる認証。
- **ユーザー登録機能**: 初回利用者がアカウントを作成できる最小限のフォーム。
- **Spring Securityの実装**: セッションベースの認証（開発効率優先）。

### Phase3. 単語管理（基本機能）

- **手動入力（一語ずつ）**: 単語、意味、カテゴリの登録・編集・削除。
- **カテゴリ管理**: ユーザーごとのカテゴリ作成・一覧表示。
- ※ CSV取込は工数削減のため次フェーズへ。

### Phase4. クイズ（テキスト形式）

- **一問一答（文字入力）**: 全カテゴリ、または選択したカテゴリからのランダム出題。
- **正誤判定**: 入力された文字列の完全一致による判定。
- ※ 音声入力・フォールバック機能はUI/UXの複雑性を避けるため次フェーズへ。

### Phase5. 学習記録・AI連携（コア価値）

- **履歴蓄積**: ユーザーID、単語ID、正誤、解答日時をDBに保存。
- **結果画面**: 解答一覧の表示。
- **偉人の名言**: クイズ終了後にランダムで1つ表示。
- **Geminiフィードバック**: 今回の正答率と、過去の苦手単語リストをプロンプトに組み込み、Gemini 3 Flashに励ましコメントを生成させる。

## 2.3 見送る範囲 (Out of Scope / Phase 2)

- **CSV一括取り込み機能**: ファイルアップロードおよびパース処理の複雑回避。
- **音声入力解答・聞き直し機能**: ブラウザ依存の音声認識実装およびリトライロジックの構築。
- **詳細な統計グラフ表示**: AIによるテキストアドバイスに注力し、ビジュアル統計は後回し。
